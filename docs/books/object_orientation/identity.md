# いまいちピンとこなさそうな議論
2つのデータが他から見て全く見分けがつかないにも関わらず、確かにそこに2つデータがあるといえるなら、かなりオブジェクト指向っぽい。
例えば、一方は変形してもう一方は変形しない操作ができたりするから。
ただし、データのコピーの場合も似たような挙動をしている？

その点で行くと、ある対象の出自や自己の識別ができるとき、データではなくてオブジェクトっぽさがある。

> [!warning]
> なんか怪しい議論（本当に全くではないので。）

## 具体例
例えば構造体のある手続き型の言語でこんなものを考える。
```
x = { l: 0 };
y = { l: 0 };
x.l = 1;
print(y.l); // => 0
```
これが `0` を出力するのはどうしてかというと、異なる変数を参照しているから。
異なる変数にたまたま同じデータが入っているだけで、 `x` に入っている `{l: 0}` と `y` に入っている `{l: 0}` を取り違えることがない。

```
x = { l: 0 };
y = x;
x.l = 1;
print(y.l); // => 0
```

この場合も、 `y = x` で `x` に入っているデータをコピーして `y` に新しくデータを入れた。
`x` の方のデータをいじっているので `y` は変わらない。
この性質（"どの"データをいじるべきかがわかっている）は、 `{l: 0}` というデータにはなくて、変数が管理している。

```
x = { l: 0 };
y = { l: 0 };
if (random()) {
  z = x;
} else {
  z = y;
}
print(z.l); // 0
```
この場合、「 `z` に入っている `{l: 0}` がどっちの分岐で代入されたのか」という考え方は意味があるが、
「 `z` に入っている `{l: 0}` がどっちの `{l: 0}` か」という考え方は、
「 \(1\) つのリンゴと \(1\) つのバナナを朝食に食べました」という文について、
これに現れる 2 つの \(1\) を入れ替えた文章を区別できるかを考えているようなもの。

逆にオブジェクト指向の場合は、「このリンゴとこのリンゴのうち、腐っているのはこのリンゴです」みたいな文章を指さしながら言ったとして、
指差しを逆にしたら別の事象を指していることになる。特定のリンゴについての話をしているときっぽい。

この点で行くと、最初に言った「2つのデータが他から見て...」の部分は確かにオブジェクト指向特有の話をしていて、
ある単語が文章の中に複数出てきていることと、その単語が別のものを指していることを区別している。

## 名前の束縛とオブジェクトの扱い方の難しさ
典型的にはこういうことが起きる。
```
x = new Counter(0);
y = x;
x.set(1);
// y.get() == 1
```
これは、 `x = y` では `x` も `y` も同じオブジェクトの別の呼び方を用意しただけなので、
扱っているオブジェクトは同じだから。（「名札モデル」というらしい。）
上のコードは以下のコードとは違う
```
x = new Counter(0);
y = x;
x = new Counter(1); // 別のオブジェクトを作成した
// y.get() == 0
```

他にも、整数はオブジェクトなのでこういうことも起きる。
```
x = new 1;
y = x;
x.set(2);
// y.get() == 2;
```

> [!Note]
> オブジェクト指向のことを考えると、これは言語では防ぐ意味がなくて、
> やるならコード規約として、こういうことが起きないように見張るしかない？

まあ、整数みたいなものは primitive といってオブジェクトにしないことがある。
あまりオブジェクト指向を徹底しない方が主流？

## 同じ content を持つ異なるオブジェクトを作る。
"状態" だけほしくて identity は別のものとして新たに作り出したいとき。
いわゆる clone を行いたい。
```
x = new Counter(0);
y = clone(x);
x.set(1)
// y.get() == 0
```
別のオブジェクトを保持しているとどう動くか決まらない。
```
a = new Counter(0);
x = new Point { u: a };
y = clone(x);
x.u.set(1);
// y.u.get() == 1 ?
```
状態として保持している全てのオブジェクトを再帰的に clone する"ような"動作を deep copy といい、
とりあえず一階層だけさらっと見て clone する"ような"動作を shallow copy という。
clone がそもそもどう動くかを考えると、
```
x = new Counter { count: 0 };
// y = clone(x);
y = new Counter { count: x.get() };
```
みたいになっている？
