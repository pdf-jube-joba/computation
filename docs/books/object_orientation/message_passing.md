オブジェクトが互いに通信しあうという世界観？
アクターモデルとか。
- 現実の言語観（自然言語でもプログラミング言語でも）だと、主語述語とか、呼び出す側みたいなのがある。
- 時間に対してどう動くか？何が起きたらオブジェクトが動くのか？どの受取順で処理されるのか？
  - 決定的である必要はなくて、何が保証されるのかが知りたい。

プロセス計算とかはオブジェクトっぽさはさておき、異なる主体の通信を考えている。
（並列性が入ってくるが。）
また結果として、相手がどのような行動をするかを観測することはできない。
どのようなデータ構造をしているかはわからないが、どんな関連した関数を呼び出せるかはわかっている。

本場のメッセージパッシングでは、他のオブジェクトにメッセージを送ることはそのオブジェクトに制御が移ることとは異なる気がする。
そのため、
```
make a {
  hey => {
    stdin print "hello";
  }
}

make b {
  hey => {
    a hey
    stdin print "world!"
  }
}
```

みたいなときに、`hello world!` の順に表示されることは保証されないかもしれない。

いずれぬせよ、オブジェクトの詳細を知ることは全く必要じゃない。
